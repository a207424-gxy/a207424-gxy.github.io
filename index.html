Lab Work

Five Labs Journey

Each lab built on the last — from a static UI to a fully persistent, database-driven application.

01

UI Layout

Jetpack Compose · Basic Screens
Reflection

In Lab 1 I built the foundational UI for SecondLife using Jetpack Compose, learning how composables and state hoisting replace traditional XML layouts. Setting up a clean screen hierarchy early saved considerable refactoring time in later labs. This lab taught me to think in components before writing a single line of logic.

02

Interaction

State · Click Handlers · Forms
Reflection

Lab 2 introduced reactive state management with remember and mutableStateOf, enabling the app to respond to user inputs dynamically. I implemented form validation for the grocery entry screen and learned how unidirectional data flow keeps UIs predictable. Handling edge cases — empty fields, invalid dates — sharpened my defensive programming skills.

03

Material Design

Material 3 · Theming · Components
Reflection

Applying Material 3 brought a consistent visual language to the app — colour schemes, typography scales, and motion semantics aligned with Google's design system. I customised the green palette to reinforce the SDG 12 theme throughout. Learning to use MaterialTheme tokens rather than hard-coded values made future design iterations far easier.

04

Navigation

Navigation Compose · Back Stack
Reflection

Navigation Compose replaced my earlier ad-hoc screen switching with a declarative, type-safe route graph. Managing the back stack and passing arguments between screens — especially expiry-date detail views — revealed the importance of a well-planned navigation architecture. This lab laid the groundwork for a multi-screen Project 1 delivery.

05

Room Database

Room · DAO · Flow
Reflection

Room transformed SecondLife from a stateless prototype into a genuinely persistent application. Defining entities, DAOs, and a typed database class clarified how Android's local storage layer works under the hood. Observing live data via Flow and surfacing it through the ViewModel meant the UI always stayed in sync with the database without manual refresh calls.

Capstone Work

Projects

Two milestone deliverables that brought all lab skills together into a production-ready Android application.

Project 1 · Lab 4 + Working Features

SecondLife

Households often waste food because they lose track of expiry dates. This app provides a localised UI to track daily groceries and alert users before items expire.

Lessons Learned

Successfully established the MVVM architecture and shared state flows across multiple screens using Navigation Compose. Separating UI, business logic, and data layers from the outset made the codebase maintainable and easy to extend. I also learned how to handle lifecycle-aware components to prevent memory leaks in long-running coroutines.

Project 2 · Lab 5 + API + Sensors + Cloud

SecondLife

To further reduce waste, users need external product data and a way to share surplus food with the community. This version integrates APIs, hardware sensors, and cloud storage.

Lessons Learned

Overcame complex integration challenges bridging Room (local), Retrofit (APIs), hardware sensors (CameraX), and Firebase Firestore (Cloud) into one cohesive flow. Debugging the synchronisation between local and remote data sources highlighted the importance of a single source of truth in the ViewModel layer. This project solidified my understanding of coroutine scopes, error handling, and offline-first design patterns.

Learning Journey

Overall Reflection

From composing a basic UI in Lab 1 to integrating Room, Retrofit, CameraX, and Firebase in Project 2, this course reshaped how I approach software development — always starting with architecture before aesthetics. Each lab felt like unlocking a new layer of Android's ecosystem, and every bug I fixed deepened my understanding of the platform far more than any tutorial could. Building SecondLife around SDG 12 gave the technical work real purpose: the app does not just demonstrate skills, it genuinely helps households make smarter, less wasteful choices. I leave this course confident in Jetpack Compose, MVVM, and cloud integration, and eager to carry these skills into real-world projects.
